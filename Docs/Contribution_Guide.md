# Contribution Guide

## Standards
- Clear PRs with scope and tests.
- Document any API or schema changes.
- Use meaningful commit messages.

## Workflow
- Fork/branch naming conventions.
- Open issues with reproducible steps.
- Code reviews required; CI must pass.

## Testing
- Unit for pipelines; integration for APIs.
- Load tests for retrieval and quiz selection.

## Security
- No secrets in repo; use environment variables.
- Report vulnerabilities via security policy.
